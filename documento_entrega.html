3. Implementação do Banco de Dados

Banco utilizado: SQLite (via sql.js) — banco relacional sem instalação de servidor, criado automaticamente na primeira execução. O arquivo 001_schema.sql contém a versão PostgreSQL completa para deploy em produção.

3.1 Script de Criação das Tabelas

001_schema.sql
-- ================================================================
--  BESTIES PET STORE — Schema Principal (SQLite / PostgreSQL)
-- ================================================================

PRAGMA foreign_keys = ON;

-- CATEGORIAS — espelha o array CATEGORIES do frontend
CREATE TABLE IF NOT EXISTS categories (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  name       TEXT NOT NULL,
  slug       TEXT UNIQUE NOT NULL,   -- "dogs", "cats" — usado no front
  emoji      TEXT,
  gradient   TEXT,                    -- gradiente CSS do card
  created_at TEXT DEFAULT (datetime('now'))
);

-- PRODUTOS — espelha a interface Product do App.tsx
CREATE TABLE IF NOT EXISTS products (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  title       TEXT NOT NULL,
  price       REAL NOT NULL CHECK (price >= 0),
  emoji       TEXT,
  bg          TEXT,                    -- cor hex: "#FFF0EB"
  badge       TEXT,                    -- "Best Seller", "New", "50% Off"
  category_id INTEGER NOT NULL REFERENCES categories(id),
  active      INTEGER NOT NULL DEFAULT 1,  -- soft delete
  created_at  TEXT DEFAULT (datetime('now')),
  updated_at  TEXT DEFAULT (datetime('now'))
);

-- USUÁRIOS
CREATE TABLE IF NOT EXISTS users (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  name          TEXT,
  email         TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,          -- hash bcrypt
  role          TEXT NOT NULL DEFAULT 'customer'
                  CHECK (role IN ('customer', 'admin')),
  created_at    TEXT DEFAULT (datetime('now')),
  updated_at    TEXT DEFAULT (datetime('now'))
);

-- PEDIDOS — order_ref mantém o formato "BST-<timestamp>" do frontend
CREATE TABLE IF NOT EXISTS orders (
  id             INTEGER PRIMARY KEY AUTOINCREMENT,
  order_ref      TEXT UNIQUE NOT NULL,
  user_id        INTEGER REFERENCES users(id),
  total          REAL NOT NULL CHECK (total >= 0),
  status         TEXT NOT NULL DEFAULT 'pending'
                   CHECK (status IN ('pending','confirmed','shipped','delivered','cancelled')),
  customer_name  TEXT,
  customer_email TEXT,
  created_at     TEXT DEFAULT (datetime('now')),
  updated_at     TEXT DEFAULT (datetime('now'))
);

-- ITENS DO PEDIDO — snapshot: preserva title/price no momento da compra
CREATE TABLE IF NOT EXISTS order_items (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  order_id   INTEGER NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  product_id INTEGER REFERENCES products(id),
  title      TEXT NOT NULL,
  price      REAL NOT NULL CHECK (price >= 0),
  qty        INTEGER NOT NULL CHECK (qty > 0),
  emoji      TEXT
);

4.2 Endpoints da API — CRUD Completo para 3 Entidades

MétodoRotaAcessoDescrição
Autenticação
POST/api/auth/registerpúblicoCadastro de usuário com hash bcrypt
POST/api/auth/loginpúblicoLogin — retorna token JWT (24h)
Categorias — Entidade 1 (CRUD completo)
GET/api/categoriespúblicoLista todas as categorias
GET/api/categories/:idpúblicoBusca categoria por id
POST/api/categoriesAdmin JWTCria nova categoria
PUT/api/categories/:idAdmin JWTAtualiza categoria
DELETE/api/categories/:idAdmin JWTRemove categoria
Produtos — Entidade 2 (CRUD completo)
GET/api/products?search=&category=públicoLista produtos com filtros de busca e categoria
GET/api/products/:idpúblicoBusca produto por id
POST/api/productsAdmin JWTCadastra novo produto
PUT/api/products/:idAdmin JWTAtualiza produto
DELETE/api/products/:idAdmin JWTDesativa produto (soft delete — active = 0)
Pedidos — Entidade 3 (CRUD completo)
GET/api/ordersAdmin JWTLista todos os pedidos com contagem de itens
GET/api/orders/:refpúblicoBusca pedido com itens e subtotais
POST/api/orderspúblicoCria pedido + itens (transação atômica)
PATCH/api/orders/:ref/statusAdmin JWTAtualiza status do pedido
DELETE/api/orders/:refAdmin JWTRemove pedido
